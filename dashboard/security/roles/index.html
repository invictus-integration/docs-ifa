Available flow permissions​

The following table describes the permitted flow-related capabilities for users, based on the set role they receive on a folder. Reader, Operator or Folder admin. You can assign both local users and groups to folders.

Allowed    Not allowed

ReaderOperatorFolder adminDescription
FlowView flow statusesNavigate to the homepage and viewing the status summary of the flows for the user.
FlowResubmit/Resume flowRun operations on the flow like Resubmit/Resume/Ignore.
FlowAdd flowCreate a new flow in a folder, possibly with alerts.
FlowEdit flowEdit flow's mapping, properties, alerts.
FlowDelete flowDelete flow from a folder.
MessagesView messagesSearch for messages went through a flow.
FolderView foldersView folder list with flows.
FolderEdit folderEdit folder's name and position in the list.
FolderGrant flow permissionsEdit folder permissions for users and groups to access the flows.
FolderDelete folderDelete folder from the list (removes all flows and sub-folders under it as well).
System admins

Users with the System admin user role (either as a local user or via a member of a group with such role) has the same permissions as the Folder admin permission on all flows in the Dashboard -- regardless whether you assigned the System admin user to the folder or not.
