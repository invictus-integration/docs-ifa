  3. your_folder •••
  4. Create New Flow

As business user, you can create new flows to track your integrations. By specifying the right mapping and context, you can track the messages going through your integrations

PropertyDescription
Flow NameA Flow Name that describes the client interaction (example: invoice request, invoice approval)
FolderThe Folder where to add the flow. Flows are always structured in folders.
MappingThe information to identify messages going through this flow. The Domain, Service, Action and Version are special business properties to 'map' messages to flows. You can use Workflow name in cases where the message goes through an Azure Logic App workflow.
Business PropertiesMetadata context properties to further describe the flow. Every Business property name you add here gets filled if the message contains the corresponding value.
Advanced Settings
  • Connected Dashboard (= hybrid setups require a link from one Dashboard (BizTalk) to the other Dashboard (Azure))
  • Show milestone and event text (= displays the flow milestone and event text data)
  • Show Resubmit / Resume (= make the 'Resubmit/Resume' flow action buttons available per flow, New since v6.2)
  • Custom resubmit URL Your resubmit URL and Custom resume URL Your resume URL (= turn these options on and provide the URLs to use your own resume and resubmit logic)

Allow access to flow

You can allow both active Microsoft Entra ID groups and local users to access the flows in the folder. Based on the allowed role, the user receives their allowed permissions on the flow.

Add an alert for a flow

You can create Azure Monitor alerts right from the flow. This allows you to pro-actively monitor your flows and get notified when something needs your attention, for example when a flow becomes inactive or starts failing.

require role assignments after installation

The Invictus Dashboard requires additional role assignments to add alerts to flows.

PropertyDescription
Alert nameThe Alert name for the alert rule must be unique per flow. (Flow name becomes a prefix.)
DescriptionThe Description text giving more context to the alert.
SeverityThe level of Severity of the alert. This determines the importance and urgency of the alert.
Schedule
  • Frequency (= how often to execute the rule - in the range of 5-1440)
  • Time window (= range that Azure uses, the logs for the last x minutes - in the range of 5-2880)
TriggerThe Threshold value and the Threshold Operator defines when to trigger the alert rule.
ThrottlingThe Suppress alerts and Suppress alerts (minutes) allows you to throttle triggered alerts for x minutes to prevent alert fatigue.
Query
  • Invictus Activity Check
  • Invictus Error Check
  • Azure Result Count

Monitors flow activity via Azure logs (custom events with name: InvictusImportJobFlowActivityAlert). Invictus logs one activity per active flow every hour. If an alert running every 60 minutes finds no logs in the last 60 minutes (threshold < 0), Invictus flags the flow as inactive and triggers an alert.

logging frequency

Logging happens per flow (not per message) and may exceed one entry per hour due to scaling or multi-threading. Reduce FlowActivityIntervalInMinutes to increase logging frequency.

Monitors flow activity via Azure logs (custom events with name: InvictusImportJobFlowErrorAlert). Invictus logs an entry for each flow that triggers an error. If an alert running every 10 minutes finds more than 0 logs in the last 10 minutes (threshold > 0), Invictus flags the flow as failed and triggers an alert.

Use custom Azure alert syntax, see the official Microsoft documentation to learn more about alerting.
