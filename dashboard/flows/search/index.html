Search for messages in a flow

  1. Your Flows
  2. Click on a flow

The Dashboard allows users to search for messages that went through a flow?. This helps during defect localization or verification of client integrations. The [Select Flows] dropdown provides you to select one or more flows for which the search query should take place.

Additionally, the Dashboard provides Basic and Advanced search capabilities to deepen the search.

  • Basic search
  • Advanced search

The user is able to perform a basic text search through the top search bar.

This searches (case-insensitive) through these properties of the flow trace.

Flow trace propertyDescription
Chain IDShows only the flow traces within the same transaction, linked together with the Chain ID, also known in Azure Logic Apps as the 'client tracking ID'.
DSAVShows only flow traces with the same mapping: Domain, Service, Action, Version. (These are special business properties that are treated differently during flow trace importing, loaded from 'tracked properties, for instance, when using importing them via Logic Apps.)

Click the [Search] button or press [Enter] to perform your query.

An additional set of filters are available when clicking open the Advanced Search pane to further subset the flow traces search query.

The following filters may be applied:

Filter parameterRequiredDescription
Date filteryesShows only the flow traces within a given time frame. Possible options are:
  • all (= last 30 days)
  • after [your date]
  • before [your date]
  • between [your start date] and [your end date]
  • last [your amount] [minutes|hours|days|weeks|months]
Chain IDnoShows only the flow traces within the same transaction, linked together with the Chain ID, also known in Azure Logic Apps as the 'client tracking ID'.
StatusnoShows only the flow traces at a given state:
  • Active: currently processing, (including resubmit & resume for Logic Apps)
  • Completed: successfully finished processing.
  • Error: failure during processing (including Logic Apps failures)
  • Ignored: when importing via LogicApps, flows that users ignored via the Dashboard.
DSAVnoShows only flow traces with the same mapping: Domain, Service, Action, Version. (Invictus treats these special business properties differently when mapping imported messages, loaded from 'tracked properties, for instance, when using importing them via Logic Apps.)
An extra Contains Word option is available to also include partial matches of these mappings.
PropertiesnoShows only flow traces that has certain custom user properties defined. You can filter by the property value by using a format:
  • Equal: include exact matches.
  • Not Equal: include differences
  • Starts With: include matching prefix
  • Ends With: include matching postfix
  • Contains: include partial matches.
  • Regex: include matching patterns (regular expression language)
