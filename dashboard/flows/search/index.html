Search for messages in a flow

  1. Your Flows
  2. Click on a flow

The Dashboard allows you to search for messages that went through a flow. This helps with defect localization or verification of client integrations. With the Select Flows dropdown you can select one or more flows to search for related messages.

  • Basic search
  • Advanced search

You can use the top Search box to perform a basic query for messages in the selected flows. This searches through the properties of the message.

Message propertyDescription
Chain IDIncludes messages with a matching Chain ID (= transaction ID), also known in Azure Logic Apps as the 'client tracking ID'.
DSAVIncludes messages with a matching mapping (Domain, Service, Action, Version).

A set of filters are available when clicking open the Advanced Search pane to further subset the flow traces search query.

FilterDescription
StatusShows only the messages at a given state:

  • Active (= messages currently processing, including resubmit & resume for Logic Apps)
  • Completed (= messages successfully finished processing)
  • Error (= failure during messages processing, including Logic Apps failures)
  • Ignored (= messages that operators ignored via the Dashboard)
  • Cancelled (= messages that were manually cancelled by operators) New since v6.2
Date filterShows only the messages within a given time frame. Possible options are:

  • All dates (= last 30 days)
  • After your date
  • Before your date
  • Between your date and your date
  • Last amount time period
DSAVShows only messages with the same mapping: Domain, Service, Action, Version. (Invictus treats these special business properties differently when mapping imported messages, loaded from 'tracked properties, for instance, when using importing them via Logic Apps.)
A Contains Word checkbox is available to include partial matches of these mappings.
Chain IDShows only the messages within the same transaction, linked together with provided Chain ID, also known in Azure Logic Apps as the 'client tracking ID'.
PropertiesShows only messages that has certain custom user properties defined. You can filter by using a format:
  • your property Equal your value (= include exact matches)
  • your property Not Equal your value (= include differences)
  • your property Starts With your value (= include matching prefix)
  • your property Ends With your value (= include matching postfix)
  • your property Contains your value (= include partial matches)
  • your property Regex your value (= include matching patterns, regular expression language)
