C
Chain ID

An identifier that groups all instances belonging to the same end-to-end transaction. Filtering by Chain ID in the Dashboard shows every message processed as part of that single transaction, even across multiple flows.

D
DSAV

Stands for Domain, Service, Action, and Version — the four business properties Invictus uses to categorize messages. In the Dashboard search you can filter by any combination of these values to find all messages of the same integration type.

F
Flow

A named message chain — a logical grouping of related messages sharing a common origin. Flows are organized in folders and defined by a name, optional mappings, and business properties.

Flow status

The state of a as shown in the Dashboard. There are four possible states:

  • Active — the message is currently being processed.
  • Completed — the message finished successfully.
  • Error — the message was suspended, dehydrated, or terminated with a failure.
  • Ignored — an operator manually dismissed the message (typically after reviewing a failure).
  • Cancelled — an operator manually cancelled the run and halted the message processing.
Flow trace

A single tracked message instance within a . It records the full lifecycle of one execution: status, timestamps, properties, and the chain of processing steps.

Each flow trace can be searched and inspected in the Dashboard.

Folder

A container used to organize flows in the Dashboard sidebar. Folders can be nested into a hierarchy with sub-folders, giving teams a clear overview of their integration landscape.

I
Invictus Dashboardalso: Dashboard

A solution that reduces the need for in-depth Azure knowledge and allows business users to focus on the status of their integrations.

Invictus Frameworkalso: Framework

A set of components that supports commonly used integration patterns in Azure.

R
Resubmit

A Dashboard action that re-triggers a from the beginning. Resubmit restarts the first workflow in the chain, even if the original message completed successfully.

See also , which only re-runs the failed steps rather than the whole chain.

Resume

A Dashboard action that re-runs only the failed steps of a , without restarting the entire chain from the beginning. Resume is only available for messages in Error status.

See also , which always restarts from the first workflow.
