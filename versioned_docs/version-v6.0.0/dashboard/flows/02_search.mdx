# Search for messages in a flow
<Breadcrumb paths={[`Your Flows`, <><FontAwesomeIcon icon={faArrowPointer} /> Click on a flow</>]} />

The Dashboard allows you to search for messages that went through a flow. This helps with defect localization or verification of client integrations.
With the <DropDownInput label="Select Flows" /> dropdown you can select one or more flows to search for related messages.  

![Select flows dropdown multi-select](/images/v2_search-advanced3.png)

<Tabs groupId="search-type">
<TabItem value="basic" label="Basic search" default>
You can use the top <TextInput label="Search" icon={faMagnifyingGlass} /> box to perform a basic query for messages in the selected flows. This searches through the properties of the message.

![Basic search example](/images/dashboard/search-for-flow-traces-basic-search.gif)


| Message property | Description                                                                                                                                                                                                                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Chain ID**        | Includes messages with a matching Chain ID (= transaction ID), also known in Azure Logic Apps as the 'client tracking ID'.                                                                                                                                                                        |
| **DSAV**            | Includes messages with a matching mapping (**Domain**, **Service**, **Action**, **Version**). |

</TabItem>
<TabItem value="advanced" label="Advanced search">
A set of filters are available when clicking open the <span style={TableHeaderStyle}>Advanced Search <FontAwesomeIcon icon={faAngleDown} /></span> pane to further subset the flow traces search query. 

![Advanced search example](/images/dashboard/search-for-flow-traces-advanced-search.gif)

| Filter           | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Status**       | Shows only the messages at a given state:<br/><br/> <ul style={NoListStyle}><li><CheckBoxInput label="Active" checked={true} /> (= messages currently processing, [including resubmit & resume for Logic Apps](./04_import-flow-traces/import-flows-via-la.mdx))</li><li><CheckBoxInput label="Completed" /> (= messages successfully finished processing)</li><li><CheckBoxInput label="Error" /> (= failure during messages processing, [including Logic Apps failures](./04_import-flow-traces/import-flows-via-la.mdx))</li><li><CheckBoxInput label="Ignored" /> (= messages that operators ignored via the Dashboard)</li><li><CheckBoxInput label="Cancelled" /> (= messages that were manually cancelled by operators) <NewSinceBadge version="v6.2" /></li></ul>                       |
| **Date filter**  | Shows only the messages within a given time frame. Possible options are:<br/><br/> <ul style={NoListStyle}><li><FontAwesomeIcon icon={faCircleDot} /> All dates (= last 30 days)</li><li><FontAwesomeIcon icon={faCircle} /> After <DateInput /></li><li><FontAwesomeIcon icon={faCircle} /> Before <DateInput /></li><li><FontAwesomeIcon icon={faCircle} /> Between <DateInput /> and <DateInput /></li><li><FontAwesomeIcon icon={faCircle} /> Last <TextInput label="amount" /> <DropDownInput label="time period" /></li></ul>                                                                                                                                                                                                                                                  |
| **DSAV**         | Shows only messages with the same mapping: <TextInput label="Domain" />, <TextInput label="Service" />, <TextInput label="Action" />, <TextInput label="Version" />. (Invictus treats these special business properties differently when mapping imported messages, loaded from 'tracked properties, for instance, when using [importing them via Logic Apps](./04_import-flow-traces/import-flows-via-la.mdx).) <br/> A <CheckBoxInput label="Contains Word" /> checkbox is available to include partial matches of these mappings.                                                                                                         |
| **Chain ID**     | Shows only the messages within the same transaction, linked together with provided <TextInput label="Chain ID" />, also known in Azure Logic Apps as the 'client tracking ID'.                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Properties**   | Shows only messages that has certain custom user properties defined. You can filter by using a format: <ul><li><DropDownInput label="your property"/> <DropDownInput label="Equal"/> <TextInput label="your value" /> (= include exact matches)</li><li><DropDownInput label="your property"/> <DropDownInput label="Not Equal"/> <TextInput label="your value" /> (= include differences)</li><li><DropDownInput label="your property"/> <DropDownInput label="Starts With"/> <TextInput label="your value" /> (= include matching prefix)</li><li><DropDownInput label="your property"/> <DropDownInput label="Ends With"/> <TextInput label="your value" /> (= include matching postfix)</li><li><DropDownInput label="your property"/> <DropDownInput label="Contains"/> <TextInput label="your value" /> (= include partial matches)</li><li><DropDownInput label="your property"/> <DropDownInput label="Regex"/> <TextInput label="your value" /> (= include matching patterns, [regular expression language](https://learn.microsoft.com/en-us/dotnet/standard/base-types/regular-expression-language-quick-reference))</li></ul> |

</TabItem>
