# Add new flows <OnlyFolderAdminsBadge/>
Each [folder](./index.md#organizing-flows-in-folders) can hold one or more 'flows'. These are message chains. You can link an aggregation of all the messages to the same incoming message. Flows are the functional representation of the message with an optional context property name and / or optional context property value.

## Add/Edit a flow
<Breadcrumb paths={["Your Flows", "••• on a folder or flow", "Create New / Edit Flow"]} />

| Flow property           | Required | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ----------------------- | :------: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Flow Name**           |   yes    | A case sensitive name that describe the client interaction (example: ex. invoice request, invoice approval, …)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Folder**              |   yes    | The location of where to save the flow. Flows are always structured in folders.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Mapping**             |    no    | The mapping that's used to 'map' flow traces to this flow. The flow trace importing treats the DSAV mapping as special business properties. Often loaded from 'tracked properties, when using [importing them via Logic Apps](./04_import-flow-traces/import-flows-via-la.mdx). The flow trace importing adds a dedicated `WorkflowName` in those cases as well to map the flow traces of the Logic App workflow to the flow.                                                                                                                                                                                                                                                                                          |
| **Business Properties** |    no    | Context properties to describe the flow. These are only here to provide metadata and aren't used in the flow tracing.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Advanced Settings**   |    no    | <ul><li>*Connected Dashboard:* in a hybrid (BizTalk + Azure) setup, it's necessary to have a link from one Dashboard (BizTalk) to the other Dashboard (Azure) for hybrid flows. [More details on hybrid setups](../settings.md).</li><li>*Show milestone and event text:* Checking this option will display the flow milestone and event text data during flow trace searches.</li><li><NewSinceBadge version="v6.2" /> *Show Resubmit / Resume*: make the 'Resubmit/Resume' flow action buttons available per flow.</li><li>*Custom resubmit/resume URLs:* Check these options and provide the necessary URLs to use [your own resume and resubmit logic](./04_import-flow-traces/import-flows-via-la.mdx).</li></ul> |
| **Alerts**              |    no    | [See alerting](#flow-alerting)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
<details id="flow-permissions">
<summary><span>**Allow access to flow**</span></summary>

<Breadcrumb paths={["Your Flows", "Click ••• of folder", "Edit Folder Permissions"]} />

You can allow both active Microsoft Entra ID groups and local users to access the flows in the folder. Based on the [chosen role](../security/03_roles.md), the user will receive their allowed permissions on the flow. 

:::info
Users with the `[System admin]` role can access all flows in every folder available.
:::
</details>
