# Add new flows <OnlyFolderAdminsBadge/>
{/* vale Vale.Spelling = NO */}
<Breadcrumb paths={[`Your Flows`, <><i>your_folder</i> •••</>, <><FontAwesomeIcon icon={faPlus} /> Create New Flow</>]} />
{/* vale Vale.Spelling = YES */}

As business user, you can create new flows to track your integrations. By specifying the right mapping and context, you can track the messages going through your integrations

| Property                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Flow Name**           | A <TextInput label="Flow Name" /> that describes the client interaction (example: *invoice request*, *invoice approval*)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Folder**              | The <DropDownInput label="Folder" /> where to add the flow. Flows are always structured in [folders](./index.mdx#organizing-flows-in-folders).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| **Mapping**             | The information to identify messages going through this flow. The <TextInput label="Domain" />, <TextInput label="Service" />, <TextInput label="Action" /> and <TextInput label="Version" /> are special business properties to 'map' messages to flows. You can use <TextInput label="Workflow name" /> in cases where the message goes through an Azure Logic App workflow.                                                                                                                                                                                                          |
| **Business Properties** | Metadata context properties to further describe the flow. Every <TextInput label="Business property name" /> you add here gets filled if the message contains the corresponding value.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Advanced Settings**   | <ul><li><ToggleInput label="Connected Dashboard" /> (= [hybrid setups](../settings.mdx) require a link from one Dashboard (BizTalk) to the other Dashboard (Azure))</li><li><ToggleInput label="Show milestone and event text" /> (= displays the flow milestone and event text data)</li><li><ToggleInput label="Show Resubmit / Resume" /> (= make the 'Resubmit/Resume' flow action buttons available per flow, <NewSinceBadge version="v6.2" />)</li><li><ToggleInput label="Custom resubmit URL" /><TextInput label="Your resubmit URL" /> and <ToggleInput label="Custom resume URL" /><TextInput label="Your resume URL" /> (= turn these options on and provide the URLs to use [your own resume and resubmit logic](./04_import-flow-traces/import-flows-via-la.mdx))</li></ul> |
| **Alerts**              | <DetailsLink targetId="flow-alerting">See alerting</DetailsLink>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

<details id="flow-permissions">
<summary><span>**Allow access to flow**</span></summary>

You can allow both active Microsoft Entra ID groups and local users to access the flows in the folder. 
Based on the [allowed role](../security/03_roles.mdx), the user receives their allowed permissions on the flow. 

{/* vale Vale.Spelling = NO */}
<Breadcrumb paths={[`Your Flows`, <><i>your_folder</i> •••</>, <><FontAwesomeIcon icon={faUserGroup} /> Edit Folder Permissions</>]} />
{/* vale Vale.Spelling = YES */}

:::info[Admins can access all flows]
Users with the *System admin* role can access all flows in every folder available.
:::
</details>

<details id="flow-alerting">
<summary><span>**Add an alert for a flow**</span></summary>

{/* vale write-good.TooWordy = NO */}
You can create [Azure Monitor alerts](https://learn.microsoft.com/en-us/azure/azure-monitor/alerts/alerts-overview) right from the flow.
This allows you to pro-actively monitor your flows and get notified when something needs your attention, for example when a flow becomes inactive or starts failing. 
{/* vale write-good.TooWordy = YES */}

:::warning[require role assignments after installation]
The Invictus Dashboard [requires additional role assignments](../installation/03_give_la_access.md) to add alerts to flows.
:::
