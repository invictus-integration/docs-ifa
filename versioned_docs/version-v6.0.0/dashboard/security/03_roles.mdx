# Control the access to your flows

## Available flow permissions
{/* vale Google.WordList = NO */}
The following table describes the permitted flow-related capabilities for users, based on the set role they receive on a folder. *Reader*, *Operator* or *Folder admin*. You can both single **local users** and **groups** to folders.

|          |                            | Reader | Operator | Folder admin | Description                                                                                                                                                           |
| -------- | -------------------------- | :----: | :------: | :----------: | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Flow     | View **flow** statuses     |    <DisallowIcon/>   |    <AllowIcon/>     |      <AllowIcon/>       | [Accessing the homepage](../flows/index.mdx) and viewing the status summary of the available flows for the user.                                                      |
|          | Resubmit/Resume **flow**   |   <DisallowIcon/>    |    <AllowIcon/>     |      <AllowIcon/>       | Users can resubmit/resume/ignore [messages mapped to an Azure Logic App workflow](../flows/04_import-flow-traces/import-flows-via-la.mdx) directly from the Dashboard |
|          | Add **flow**               |   <DisallowIcon/>    |    <DisallowIcon/>     |      <AllowIcon/>       | [Adding a new flow to a folder](../flows/01_add.mdx), possibly with alerts                                                                                            |
|          | Edit **flow**              |   <DisallowIcon/>    |    <DisallowIcon/>     |      <AllowIcon/>       | [Changing the flow](../flows/01_add.mdx) mapping, properties, alerts                                                                                                  |
|          | Delete **flow**            |   <DisallowIcon/>    |    <DisallowIcon/>     |      <AllowIcon/>       | Removing a flow from a folder                                                                                                                                         |
| Messages | View **messages**          |   <AllowIcon/>    |    <AllowIcon/>     |      <AllowIcon/>       | [Searching for messages](../flows/02_search.mdx) for a selected flow available for the user                                                                           |
| Folder   | View **folders**           |   <AllowIcon/>    |    <AllowIcon/>     |      <AllowIcon/>       | Viewing folder hierarchy structure with available flows for the user                                                                                                  |
|          | Edit **folder**            |   <DisallowIcon/>   |    <DisallowIcon/>     |      <DisallowIcon/>       | Renaming the folder and changing hierarchy structure                                                                                                                  |
|          | Grant **flow** permissions |   <DisallowIcon/>    |    <DisallowIcon/>     |      <DisallowIcon/>       | [Assigning permissions on folders](../flows/01_add.mdx#flow-permissions) for users and groups to access the flows                                                     |
|          | Delete **folder**          |   <DisallowIcon/>    |    <DisallowIcon/>     |      <DisallowIcon/>       | Removing a folder from the hierarchy, removing all flows and sub-folders under it as well                                                                             |

:::info[System admins]
Users with the *System admin* user role (either as a local user or via a member of a group with such role) has the same permissions as the *Folder admin* permission on <u>all</u> flows in the Dashboard -- regardless whether you assigned the *System admin* user to the folder or not.
:::

## Available user roles
The following table describes what non-flow related parts of the Dashboard are available for users, based on their assigned roles: *Non-admin* and *System admin*. Microsoft Entra ID users receive their role based on the role of the group that they are a member of. Local users receive personalized roles for each one separately. 

|          |                                 | Non-admin | System admin | Description                                                                                                                                                                                    |
| -------- | ------------------------------- | :-------: | :----------: | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Users    | View **users**                  |     <DisallowIcon/>     |      <AllowIcon/>       | Accessing the `[Users]` page and viewing all **local** and **Entra ID** users                                                                                                                  |
|          | Add **local user**              |     <DisallowIcon/>     |      <AllowIcon/>       | Adding a new **local** user with credentials to the Dashboard                                                                                                                                  |
|          | Add **Microsoft Entra ID user** |     <PartialAllowIcon/>     |      <PartialAllowIcon/>       | 'Sign in with' on the sign in page automatically creates an user, when that user is a member of a [synced group](../installation/01_give_ad_access.mdx)                                        |
|          | Edit **local user** role        |     <DisallowIcon/>     |      <AllowIcon/>       | Changing the user role of **local** users (**Entra ID** users receive their role via their **group**)                                                                                          |
|          | Delete **user**                 |     <DisallowIcon/>     |      <AllowIcon/>       | Removing a **local** or **Entra ID** user from accessing the Dashboard permanently (**Entra ID** users will need to removed from synced groups as well, if they're not allowed to log back in) |
| Groups   | View **groups**                 |     <DisallowIcon/>     |      <AllowIcon/>       | Accessing the `[Groups]` page and viewing all synced groups                                                                                                                                    |
|          | Sync **groups**                 |     <DisallowIcon/>     |      <AllowIcon/>       | Running the [`[Sync all groups]`](../installation/01_give_ad_access.mdx) operation                                                                                                             |
|          | Disable **group**               |     <DisallowIcon/>     |      <AllowIcon/>       | [Temporary pause the group's authority](../installation/01_give_ad_access.mdx) from accessing flows                                                                                            |
|          | Edit **group** role             |     <DisallowIcon/>     |      <AllowIcon/>       | [Changing the user role](../installation/01_give_ad_access.mdx) of a single group                                                                                                              |
|          | Delete **group**                |     <DisallowIcon/>     |      <AllowIcon/>       | Removing a group's authority from accessing flows (if a Microsoft Entra ID user is still a member of the group, the group still syncs)                                                         |
| Settings | View **settings**               |     <AllowIcon/>     |      <AllowIcon/>       | Accessing the [`[Settings]`](../settings.mdx) page and viewing all Dashboard settings                                                                                                          |
|          | Edit **settings**               |     <DisallowIcon/>     |      <AllowIcon/>       | [Changing the Dashboard settings](../settings.mdx)                                                                                                                                             |
| Audits   | View **audits**                 |     <DisallowIcon/>     |      <AllowIcon/>       | Accessing the [`[Audits]`](../security/04_auditing.mdx) page and viewing all security audits                                                                                                   |
