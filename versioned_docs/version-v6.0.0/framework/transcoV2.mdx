:::note[motivation]
Integration scenarios often need to 'promote' or 'enrich' messages/data from either internal or external data sources. Codit is a long-term implementer of something called 'Transco', which was originally a Microsoft BizTalk component. Now, modern Azure Logic Apps integration benefits from this behavior.

Invictus provides a Transco component to promote properties from a SQL database or to transform XML content via XSLT transformations. The component can also promote values to the context of the response.
:::

## Component usage
The Transco component has endpoints available for XML or JSON content.
Given a stored Transco configuration file, the component promotes values from a client's SQL database or transforms XML content via a stored XSLT file.

## Transco configuration file
The component requires a JSON Transco configuration file to describe the instructions to perform. Execution of instructions happens in the order in which they appear.
Store the configuration file in the Azure Storage Account in the `Transcov2configstore` container, in the `/Configs` folder.

<Tabs groupId="Transco-instructions">
<TabItem value="sql-command" label="SQL Commands" default>

<Walkthrough>
<Collapsible title="Set the scope of the command instruction">
By default, the Transco component applies the command instruction to the entire `Content`.
You can specify a scope for the instruction by providing a `scopePath` in the configuration file.
This path is an XPath or JPath expression that points to a specific part of the content. The command applies then only to that part of the content.

```json
"scopePath": [XPath/JPath of content scope],
"namespaces": [
  {
    "namespace": [XML Namespace],
    "prefix": [XML Namespace prefix]
  }
]
// ...
```

</Collapsible>

<Collapsible title="Set the connection to the SQL database">
Provide the database connection via a raw connection string or the name of an Azure Key Vault secret holding the connection string.

```json
"command": {
   "databaseConnectionString":[Raw connection string to DB],
   "databaseKeyVaultName": [Name of DB connection string secret in Key Vault],
}
// ...
```
</Collapsible>

<Collapsible title="Set the SQL query and parameters">

You can populate the parameters of the SQL query from these sources:
 - `path` (= XML/JSON request content via XPath/JPath)
 - `context`(= Request `Context` key-value collection)
 - `value` (= Fixed value defined in the config file itself)
<br/>

```json
"command": {
  "commandValue": [SQL query to execute],
  "parameters": [
    {
      "paramName": [Name of parameter in SQL query],
      "value": [XPath/JPath if "valueType=path", Context key if "valueType=context", or any string value if "valueType=fixedValue"],
      "type": [SQL DB type. See: https://learn.microsoft.com/en-us/dotnet/api/system.data.sqldbtype],
      "valueType": ["path" or "fixedValue" or "context"]
    },
    {
      "paramName": [Name of parameter in SQL query],
      "value": [XPath/JPath if "valueType=path", Context key if "valueType=context", or any string value if "valueType=fixedValue"],
      "type": [SQL DB type. See: https://learn.microsoft.com/en-us/dotnet/api/system.data.sqldbtype],
      "valueType": ["path" or "fixedValue" or "context"]
    }
  ]
  // ...
}
```

Denote parameters in the SQL query with an `@` symbol. You may give parameters a name or indexed with a number.
```sql
SELECT CustomerStatus FROM dbo.Customers WHERE CustomerName = @Name AND Active = @Active*
-- or
SELECT CustomerStatus FROM dbo.Customers WHERE CustomerName = @1 AND Active = @2*
```

:::tip[control SQL query result]
With the `isMandatory` option, you control whether a `null` result of the SQL query should throw an error.
You can also specify a `defaultValue` to handle those cases.

When the query returns more than one column, you use the `columnName` of the value to be inserted in the content.

```json
"command": {
  // ...
  "isMandatory": [If true, will throw error when result is null],
  "defaultValue": [Default value of result],
  "columnName": [Obtain value from specified column if query returns more than one field. If empty, use value from first column],
}
```
:::

</Collapsible>

<Collapsible title="Set the destination of the SQL query result">

The `destination` is an XPath/JPath expression that navigates within the `Content` to insert the result of the SQL query.
For a XPath ending at an attribute (ex: `/persons/person/@name`), the component inserts the value into that attribute.
Otherwise, the component inserts the value in the inner text of the XML node.

You can also promote the SQL query result to the `Context` instead of the `Content`. In that case, set `promoteToContext=true`.

```json
"command": {
  // ...
  "destination": [XPath/JPath of the results destination, or Context key if "promoteToContext=true"],
  "promoteToContext": [If true, the component saves the SQL query result at the Context with "destination" as key]
}
```

</Collapsible>
</Walkthrough>
</TabItem>

<TabItem value="xml-transformation" label="XML Transformation">

<Walkthrough>
<Collapsible title="Specify name of the XSLT transformation file">

The Transco component can execute XSLT transformations on XML content. Specify the name of the XSLT file as an instruction in the Transco configuration file.
Store the XSLT file in the Azure Storage Account in the `Transcov2configstore` container, in the `/XSLTs` folder.

```json
{
  "instructions": [
    {
      "xsltTransform": "[Name of XSLT file]"
    }
  ]
}
```
</Collapsible>
<Collapsible title="Specify required assemblies and dependencies">
When the XSLT transformation requires custom extension functions, specify them as required assemblies and dependencies in the Transco configuration file.
Store the assembly and dependency DLL files in the Azure Storage Account in the `Transcov2configstore` container, in the `/Assemblies` folder.

```json
{
  "instructions": [
    {
      "xsltTransform": "[Name of XSLT file]",
      "extensions": [
        {
          "namespace": "[Assembly namespace]",
          "assemblyName": "[Assembly name]",
          "className": "[Assembly class name]",
          "dependencies": [
            "[DLL dependency file name]",
            "[DLL dependency file name]"
          ]
        }
      ]
    }
  ]
}
```

</Collapsible>
</Walkthrough>
</TabItem>

<TabItem value="matrix-basic" label="Matrix basic promote">

The basic promote is a way of promoting values to the context. This version accept a list of parameters and promotes them to the Context. <u>The component only transfers found values to the response.</u>

:::tip
Transco XML/JSON operations also support promotion of values to the context.
* Set the option `promoteToContext=true` in the Transco configuration file.
* Set the option `destination` to the key of the desired destination in the context. 
The SQL operation will now save the result to the context.
:::
